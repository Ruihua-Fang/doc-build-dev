مقدمه

به دوره‌ی آموزشی هاگینگ‌فیس خوش آمدید! این مقدمه شما را در طی مراحل راه‌اندازی محیط کار راهنمایی خواهد کرد. اگر تازه این دوره را شروع کرده‌اید، پیشنهاد می‌کنیم ابتدا به فصل اول نگاهی بیاندازید و سپس به اینجا برگردید و محیط کار را راه‌اندازی کنید تا بتوانید خودتان کد را اجرا کنید.

تمام کتابخانه‌هایی که ما در این دوره استفاده می‌کنیم پکیج‌های پایتون هستند. از این رو در اینجا به شما نشان خواهیم داد چگونه محیط کار پایتون را راه‌اندازی نموده و کتابخانه‌های مورد نیاز را نصب کنید.

ما دو را راه‌اندازی محیط کار را نشان خواهیم داد: استفاده از نوت‌بوک کولَب و استفاده از محیط مجازی پایتون. می‌توانید هرکدام را که می‌خواهید انتخاب کنید. برای تازه‌کارها توصیه مؤکد داریم که از نوت‌بوک کولَب استفاده کنند.

توجه کنید که به محیط ویندوز نخواهیم پرداخت. اگر از ویندوز استفاده می‌کنید توصیه می‌کنیم از نوت‌بوک‌های کولَب استفاده کنید. اگر از سیستم‌عامل مک یا توزیعی از لینوکس استفاده می‌کنید می‌توانید از هر‌کدام از راه‌هایی که در اینجا اشاره می‌کنیم استفاده کنید.

برای طی بخش زیادی از این دوره نیاز به حساب کاربری ‌هاگینگ‌فیس‌ دارید. پیشنهاد می‌کنیم همین الان حساب خود را بسازید.

استفاده از نوت‌بوک‌ کولَب گوگل

استفاده از نوت‌بوک کولَب ساده‌ترین راه شروع است. با مرورگر خود یک نوت‌بوک جدید باز کنید و بلافاصله شروع به کد زدن کنید!

اگر با کولَب آشنایی ندارید توصیه می‌کنیم از این راهنما استفاده کنید. کولَب به شما امکان استفاده از سخت‌افزار‌‌‌های شتاب‌دهنده مانند GPU یا TPU می‌‌دهد و برای پردازش‌های سبک رایگان است.

وقتی که با محیط کاربری کولَب آشنایی اولیه پیدا کردید، یک نوت‌بوک جدید بسازید و مراحل راه‌اندازی را شروع کنید.



مرحله اول نصب کتابخانه‌هایی است که در این دوره استفاده می‌کنیم. برای نصب کتابخانه‌ها از pip استفاده می‌کنیم که پکیج‌منیجر پایتون است. در نوت‌بوک‌ها می توانید فرمان‌های سیستمی را با اضافه کردن یک علامت تعجب به ابتدای خط اجرا کنید. برای نصب کتابخانه ترنسفورمرهای هاگینگ‌فیس از این فرمان استفاده کنید:

!pip install transformers

برای اطمینان از نصب درست این پکیج، آن را ایمپورت کنید:

import transformers


این فرمان نسخه بسیار کم حجمی از ترنسفورمرهای هاگینگ‌فیس را نصب می‌کند و فریمورک‌ یادگیری ماشین مشخصی مانند پایتورچ یا تنسورفلو را اضافه نمی کند. با توجه به اینکه ما از بسیاری از قابلیت‌های مختلف این کتابخانه استفاده خواهیم کرد، پیشنهاد می‌کنیم از نسخه پرحجم‌تر مخصوص توسعه نرم‌افزار که به همراه تمامی پکیج‌های وابسته برای تقریبا همه مسائل قابل تصور نصب می‌شود استفاده کنید:

!pip install transformers[sentencepiece]

اجرای این فرمان مقداری به طول می‌انجامد ولی برای طی بقیه دوره نیازی به نصب پکیج دیگری نخواهید داشت!

استفاده از محیط مجازی پایتون

اگر ترجیح می‌دهید از یک محیط مجازی پایتون استفاده کنید، اولین مرحله نصب پایتون روی سیستم‌تان است. پیشنهاد می‌کنیم از این راهنما استفاده کنید. اگر پایتون را نصب کردید، می‌توانید فرمان‌های پایتون را در ترمینال اجرا کنید. برای شروع این فرمان را اجرا کنید تا قبل از ادامه از نصب درست پایتون اطمینان حاصل کنید.

python --version

خروجی این فرمان باید شماره نسخه پایتون نصب شده روی سیستم‌تان باشد. وقتی فرمان‌های پایتون را در ترمینال اجرا می کنید نسخه “اصلی” پایتون روی سیستم خود را درگیر می کنید. توصیه می‌کنیم این نسخه را با نصب پکیج‌های مختلف شلوغ نکنید تا بتوانید با تکیه بر این نسخه خالی از هرگونه پکیج اضافه، محیط‌های جداگانه‌ای برای هر‌‌ پروژه خود درست کنید. با این روش هر پروژه می‌تواند وابستگی‌ها و پکیج‌های خود را داشته‌باشد و دیگر نگران ناهماهنگی‌های احتمالی میان پکیج‌های نصب شده برای پروژه‌های مختلف نیستید. این کار در پایتون با استفاده از محیط‌های مجازی انجام می‌شود. محیط مجازی، پوشه ای قائم به خود روی فایل‌سیستم است که محتوی نسخه‌ی خاصی از پایتون به همراه تمام پکیج‌های مورد استفاده در یک پروژه خاص است. ساخت این پوشه با ابزارهای مختلفی امکان‌پذیر است. ما در اینجا از ابزار رسمی پایتون به نام venv استفاده می‌کنیم. ابتدا پوشه‌ای جدید برای پروژه خود ایجاد کنید. برای مثال پوشه‌ای به نام transformers-course زیر پوشه خانه خودتان در سیستم‌عامل:

mkdir ~/transformers-course
cd ~/transformers-course

درون این پوشه، با استفاده از ماژول venv پایتون، محیط مجازی خود را بسازید:

python -m venv .env

انتظار می‌رود الان زیر پوشه پروژه شما تنها یک پوشه دیگر به نام .env وجود داشته باشد.

ls -a
.      ..    .env

برای ورود و خروج از محیط مجازی پروژه خود از اسکریپت‌های activate و deactivate استفاده کنید:

\# Activate the virtual environment
source .env/bin/activate

\# Deactivate the virtual environment
source .env/bin/deactivate

‌ می‌توانید با استفاده از فرمان which python از فعال شدن محیط مجازی خود اطمینان حاصل کنید:

which python
/home/&lt;user&gt;/transformers-course/.env/bin/python

نصب وابستگی‌ها مانند آنچه در بخش استفاده از گوگل کولَب گفتیم، اکنون نیاز به نصب پکیج‌های موردنیاز برای ادامه دوره را داریم. می‌توانید نسخه پکیج ترنسفورمرهای هاگینگ‌فیس مخصوص توسعه‌ی نرم‌افزار را با استفاده از پکیج‌منیجر pip نصب کنید:

pip install "transformers[sentencepiece]"

شما تمام مراحل راه‌اندازی را طی کرده و آماده شروع دوره هستید!
